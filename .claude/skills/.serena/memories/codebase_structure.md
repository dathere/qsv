# Codebase Structure

## Source (`src/`) ‚Äî 29 TypeScript files (+ 2 ambient `.d.ts`)
### Core Server
- `mcp-server.ts` ‚Äî QsvMcpServer class, main entry point, tool dispatch (`toolDispatchMap`)
- `index.ts` ‚Äî library entry point (re-exports public API)
- `config.ts` ‚Äî centralized config via `QSV_MCP_*` env vars
- `types.ts` ‚Äî shared interfaces (QsvSkill, SkillParams, McpToolResult, etc.)
- `version.ts` ‚Äî version resolution + package.json/manifest.json parity check

### Tool System
- `mcp-tools.ts` ‚Äî barrel re-export of public symbols (back-compat surface for tests + consumers)
- `tool-definitions.ts` ‚Äî MCP tool schema creation (createToolDefinition, createGenericToolDefinition)
- `tool-handlers.ts` ‚Äî tool call processing (handleToolCall, handleGenericCommand)
- `tool-constants.ts` ‚Äî pure constants (COMMON_COMMANDS, MAX_*, AUTO_INDEX_SIZE_MB, symbols)
- `command-guidance.ts` ‚Äî COMMAND_GUIDANCE map with emoji-coded hints

### Execution
- `executor.ts` ‚Äî SkillExecutor class: validates params, builds args, runs qsv
- `concurrency.ts` ‚Äî slot-based concurrency control for parallel operations
- `spawn-utils.ts` ‚Äî child process spawn utilities (kill grace period, stdout/stderr caps)

### Skill Loading
- `loader.ts` ‚Äî SkillLoader: loads/indexes JSON skill definitions from `qsv/` dir
- `bm25-search.ts` ‚Äî ToolSearchIndex: BM25 full-text search over tools

### Filesystem & Files
- `mcp-filesystem.ts` ‚Äî FilesystemResourceProvider: file listing, metadata, conversion
- `file-operations.ts` ‚Äî file path resolution, conversion args, output formatting
- `converted-file-manager.ts` ‚Äî tracks Excel/ODS ‚Üí CSV conversions (LIFO cache)
- `working-dir-manager.ts` ‚Äî working dir state, elicitation, MCP roots sync (extracted from mcp-server.ts)

### Integrations
- `duckdb.ts` ‚Äî DuckDB SQL query integration
- `parquet-bridge.ts` ‚Äî Parquet format support
- `mcp-sampling.ts` ‚Äî MCP sampling protocol support

### Utilities
- `utils.ts` ‚Äî error handling, result helpers, version comparison
- `update-checker.ts` ‚Äî checks for qsv updates
- `pipeline-manifest.ts` ‚Äî pipeline/workflow manifest (reproducibility)
- `browse-directory.ts` ‚Äî directory browsing UI
- `installer.ts` ‚Äî installation helpers
- `ui/directory-picker-html.ts` ‚Äî HTML UI for directory picker

### Ambient types
- `wink-bm25-text-search.d.ts`, `wink-nlp-utils.d.ts`

## Tests (`tests/`) ‚Äî 26 `.test.ts` files + `test-helpers.ts` + `fixtures/`
Mirror source structure: `tests/<module>.test.ts`. Notable: `qsv-integration.test.ts`, `mcp-app.test.ts`, `deferred-loading.test.ts`, `elicitation.test.ts`, `tool-filtering.test.ts`, `tsv-output.test.ts`, `hook-scripts.test.ts`.

## Skills (`qsv/`) ‚Äî 55 auto-generated JSON skill files
Named `qsv-<command>.json`, generated by `qsv --update-mcp-skills` from repo root. Plus `README.md`.

## Agents (`agents/`) ‚Äî 3 markdown agent definitions
`data-analyst.md`, `data-wrangler.md`, `policy-analyst.md`

## Skills Workflows (`skills/`) ‚Äî 15 skill directories
`bls-query`, `csv-query`, `csv-wrangling`, `data-clean`, `data-convert`, `data-describe`, `data-join`, `data-profile`, `data-quality`, `data-validate`, `data-viz`, `genai-disclaimer`, `infer-ontology`, `qsv-performance`, `reproducible-analysis`

## Scripts (`scripts/`) ‚Äî 11 build / packaging / hook scripts
`clean-bundles.js`, `cowork-setup.cjs`, `install-mcp.js`, `log-session-end.cjs`, `log-turn-summary.cjs`, `log-user-prompt.cjs`, `log-web-results.cjs`, `package-mcpb.js`, `package-plugin.js`, `qsv-utils.cjs`, `run-tests.js`

## Other top-level
- `hooks/` ‚Äî `hooks.json` for Claude Code hooks
- `docs/` ‚Äî documentation files
- `.claude-plugin/` ‚Äî plugin manifest pointing at `.mcp.json`
- `manifest.json` ‚Äî Claude Desktop extension/plugin manifest (version must match `package.json`)
- `qsv-data-wrangling-<version>.plugin` ‚Äî packaged Cowork plugin
- `qsv-mcp-server-<version>.mcpb` ‚Äî packaged MCP bundle
