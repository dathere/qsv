# Workflow Preferences

- **Commit completed work before starting additional changes.** When a unit of
  work is finished, tested, and clean, commit it before beginning the next
  change — don't let multiple logical changes pile up uncommitted. (Stated by
  the user 2026-05; e.g. commit a base feature, then commit a follow-up
  refinement as a separate commit on the same branch.)
- "auto mode" = proceed autonomously through the planned/approved work without
  pausing for per-step confirmation.
